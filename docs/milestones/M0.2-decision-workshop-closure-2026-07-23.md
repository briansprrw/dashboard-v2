## 3. Guided-review detail decisions (resolved 2026-07-23)

Brian resolved the prior OD-1..OD-3 items during the guided M0 review. They are no longer open.

### OD-1 — Recycle-state storage naming and `users` states — RESOLVED

- **Why needed:** AR §4 introduces a recycled account state and whole-account soft-delete; TA schema only has `users.state CHECK('active','disabled')` and `sheets.state CHECK('active','archived')`. The exact stored states/columns (`recycled_at`, a third user state, cascade encoding) are an M2 schema decision.
- **Constraints/evidence:** M0.1 rows L5, L6, R7.
- **Option A — rename to `recycled_at` / add `recycled` state now (in M0.3 plan text):** clearest vocabulary; slightly more M0.3 doc churn. **Consequences:** M2 schema matches product language.
- **Option B — keep `archived`/`disabled` internally, document the mapping:** less churn; risk of vocabulary drift.
- **Decision (M0-D22):** Use `recycled_at` and an explicit `recycled` account state. Implement and verify the schema in M2.

### OD-2 — Due-color band count (6 vs 7) — RESOLVED

- **Why needed:** PP task-display spec lists 6 due states; the approved mockup renders 7 (adds "soonish" 4–7 days). This is a visual/product detail for M3.
- **Constraints/evidence:** M0.1 row E3.
- **Option A — 7 bands (mockup):** matches the approved visual target; richer glance signal.
- **Option B — 6 bands (PP):** simpler legend.
- **Decision (M0-D23):** Use seven semantic bands. Soon/soonish/future thresholds are user-configurable through validated local device settings until synchronized profiles exist.

### OD-3 — Two-column display toggle vs pure automatic flow — RESOLVED

- **Why needed:** The mockup's Display-profile screen shows a "Two-column layout on wide screens" toggle, but AR §7 approves **automatic** 1–3 column flow with no manual column-count selector.
- **Constraints/evidence:** M0.1 row V1.
- **Option A — pure automatic, remove the toggle:** matches AR literally; least config.
- **Option B — keep a boolean "allow multi-column" cap (not a fixed count):** preserves a user override without reintroducing a column-count picker.
- **Decision (M0-D24):** Replace the binary toggle with local min/max column bounds from 1–3 (default 1–3). Max is firm; provisionally, min may yield at unsafe widths. M3 must report and evaluate fallback behavior.
